# Contexto TIEMPO

## Nombre del proyecto

TIEMPO Delivery.

## Objetivo general

Desarrollar desde cero una plataforma integral de delivery para clientes, restaurantes, repartidores y operadores administrativos.

## Rutas principales

- `/`: landing page publica.
- `/admin`: dashboard administrativo.
- `/app`: app movil/PWA para clientes.
- `/api`: API interna.

## Stack tecnologico

- Laravel
- MySQL
- Livewire
- Bootstrap o Tailwind
- PWA
- Capacitor para futura APK

## Modulos principales

- Pedidos
- Clientes
- Categorias
- Restaurantes
- Productos
- Repartidores
- Pagos
- Reportes
- Usuarios
- Configuracion

## Reglas generales del proyecto

- Desarrollar desde cero en Laravel.
- No reutilizar codigo, rutas, layouts, estilos ni modulos heredados.
- Trabajar por modulos.
- Antes de programar, proponer plan de archivos.
- No modificar archivos globales sin autorizacion.
- Mantener `/admin`, `/app` y `/api` claramente separados.
