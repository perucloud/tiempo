# Snippet: Conventional Commits

Formato:

```text
tipo: descripcion breve
```

Tipos comunes:

- `docs`: documentacion.
- `feat`: funcionalidad.
- `fix`: correccion.
- `refactor`: mejora interna.
- `test`: pruebas.
- `chore`: mantenimiento.

Ejemplos:

```text
docs: agregar roadmap maestro
feat: crear modulo de pedidos
fix: corregir validacion de voucher
test: agregar pruebas de checkout
```

Reglas:

- Un commit por cambio coherente.
- No mezclar modulos sin motivo.
- No incluir secretos.
